Grundlagen

Fachbegriffe

  • Daten
    • digitale gespeicherte Informationen, die automatisch und elektr. verarbeitet werden
    • werden durch den Nutzer „interpretiert“
    • alle Daten sind Zeichenketten, die nach festgelegten Regeln aufgebaut sind

  • Datenbasis(DB)

    Ist die Gesamtheit der Daten mit Struktur inklusive ihrer Beziehungen zueinander

  • Datenbankmanagementsystem(DBMS)

    Ist die Oberfläche zur Eingabe und Verwaltung

  • Datenbanksystem

    Ist die Einheit aus Datenbasis und Datenbankmanagement

  • Warum sollte man Datenbanken benutzen?
    • über den Login kann man Zugriffsrechte regeln
    • Speicherplatz
    • Beliebig viele Zeilen (wächst dynamisch)
    • Automatisches Aktualisieren von Dateneinträgen

  • Von der Idee zur Datenbank
    • Spalten sollen anders organisiert werden z.B. Vor-/Nachname
    • Einträge sollen ergänzt werden z.B. Vorwahl/Telefon
    • Angabe der Datenallgemeingültig formatieren z.B. Alter -> Geburtsdatum

    • Einführung in MC-Acces

    • Felddatentypen: kurzer/langer Text, Datum/Uhrzeit, Link, Autowert, …
    • Acces bietet:
      • Tabellen (Gesamtheit ->Tabellenschema)
      • Beziehungen (zwischen Tabellen)
      • Abfragen (zum Auswerten)
      • Formulare (zum Eingeben)
    • Tabellenaufbau
      • Primärschlüssel (Autowert, Zahl, ohne Duplikate) -> minimal, eindeutig
      • Merkmale (Spaltenüberschriften)
      • Datensatz (Zeilen der Tabelle)
      • Datenfeld (einzelne Zelle)
      • Datenfeldwert (Inhalt einer Zelle)
      • Tabellennamen
    • Felddatentypen
      • Vorschlag von Access betrachten (Integer, Text, Datum, Ja/Nein, …)
      • Nachschlage-Assistenten (Auswahlfelder, …)
      • Indiziert (als Schlüssel)
      • Gültigkeitsregeln (Eingabe wird eingeschränkt)
      • Eingabeformat (Datum: TT.MM.JJJJ)
    • Beziehungen: verknüpfen der Tabellen miteinander
      • 1:1 -> Jedem Datensatz einer Tabelle wird genau ein Datensatz einer anderen Tabelle zugeordnet und umgekehrt
      • 1: n -> Jedem Datensatz einer Tabelle wird genau ein Datensatz einer anderen Tabelle zugeordnet; umgekehrt kann ein Datensatz aber beliebig vielen Datensätzen der anderen Tabelle zugeordnet sein.
      • n:m -> Jedem Datensatz einer Tabelle werden beliebig viele Datensätze einer anderen Tabelle zugeordnet; umgekehrt werden einem Datensatz ebenfalls beliebig viele Datensätze der anderen Tabelle zugeordnet.
    • Beziehungen
      1. Beziehung
        • Direkte Ergänzung in der Ausgangstabelle
        • Hinzufügen von Tabellen
        • Schlüssel festlegen
        • Dies kannn zu einer Situation führen, in der zwei Namen existieren, dass beheben wir, in dem wir das in eine neue Tabelle auslagern
      2. Beziehung
        • 2 Tabellen werden direkt miteinander verknüpft
        • Neue Tabelle wird mit einem Fremdschlüssel, mit der anderen Tabelle verknüpft
        • Die entstandene Beziehung als Verknüpfungstabelle auslagern
      3. Beziehung
        • Merkmal hinzufügen
        • Zuordnung über Verknüpfungstabelle
  • Datenanalyse
    1. Bedarf betrachten:
      • Wofür wird die Datenbank benötigt?
      • Was soll die Datenbank leisten?
      • Welche Daten sollen erfasst werden?
      • Was soll hinterher als Prozess stattfinden?
      • Wer benutzt das?
    2. Woher kommen meine Infos
      • Alle Formulare, die bisher die Aufgabe übernommen haben, als Vorlage nutzen
      • Interview
      • Fragenkatalog
      • Alte Datenbasis nutzen
    3. Analyse der Daten
      1. Entitäten(Menge gleichartiger Objekte)
      2. Beziehungen (Verhältnis der Entitäten zueinander)
      3. Beziehungsmerkmale (Eigenschaften von Entitäten)
      4. Nominalextraktion (Methode um 1 - 4 zu bearbeiten)
        • Anforderungsliste liegt vor und wird bearbeitet
        • Suchen aller Hauptwörter/ Nomen
        • ermittle daraus die Beziehungsarten
